**Задание 00**

Создай новую таблицу `person_discounts` для хранения персональных скидок клиентов.

Таблица должна содержать:

- `id` — первичный ключ (`Primary Key`) с тем же типом данных, что и в существующих таблицах;
- `person_id` — внешний ключ (`Foreign Key`) на таблицу `person`;
- `pizzeria_id` — внешний ключ (`Foreign Key`) на таблицу `pizzeria`;
- `discount` — размер скидки в процентах (`numeric`).

Имена ограничений внешних ключей должны соответствовать шаблону:

`fk_<table_name>_<column_name>`

---

**Задание 01**

Заполни таблицу `person_discounts` с помощью конструкции `INSERT INTO ... SELECT ...`.

При заполнении:

- сгруппируй данные по `person_id` и `pizzeria_id`;
- вычисли скидку по правилам:
  - 1 заказ → `10.5`;
  - 2 заказа → `22`;
  - 3 и более заказов → `30`;
- для генерации значения `id` используй:

```sql
ROW_NUMBER() OVER ()
```

---

**Задание 02**

Напиши SQL-запрос, который возвращает историю заказов с указанием:

- имени клиента;
- названия пиццы;
- полной стоимости;
- стоимости с учетом персональной скидки (`discount_price`);
- названия пиццерии.

Отсортируй результат по имени клиента и названию пиццы.

---

**Задание 03**

Создай уникальный многоколоночный индекс `idx_person_discounts_unique`, который предотвращает дублирование пар:

- `person_id`;
- `pizzeria_id`.

Предоставь любой SQL-запрос с выводом `EXPLAIN ANALYZE`, подтверждающим использование индекса.

---

**Задание 04**

Добавь следующие ограничения (`constraints`) для таблицы `person_discounts`:

- `person_id` не должен содержать `NULL` (`ch_nn_person_id`);
- `pizzeria_id` не должен содержать `NULL` (`ch_nn_pizzeria_id`);
- `discount` не должен содержать `NULL` (`ch_nn_discount`);
- значение `discount` по умолчанию должно быть `0`;
- значение `discount` должно находиться в диапазоне от `0` до `100` (`ch_range_discount`).

---

**Задание 05**

Добавь комментарии к таблице `person_discounts` и ко всем ее столбцам.

Комментарии могут быть написаны на русском или английском языке и должны описывать назначение таблицы и каждого атрибута.

---

**Задание 06**

Создай последовательность `seq_person_discounts`, начинающуюся со значения `1`.

Настрой столбец `id` таблицы `person_discounts` так, чтобы его значение автоматически генерировалось из этой последовательности.

Значение последовательности должно быть установлено по формуле:

`количество строк в таблице person_discounts + 1`

**Запрещено:** использовать жестко заданное количество строк (`hard-coded`) при установке значения последовательности.
