
## **Задание 00**

Чтобы надежнее работать с данными и не терять события изменений, реализуй механизм аудита для операций `INSERT`.

- Создай таблицу `person_audit`.
- Возьми за основу структуру таблицы `person`.
- Добавь к ней следующие новые колонки для аудита:
  - `created` — метка времени создания события (`timestamp with time zone`), значение по умолчанию — текущая дата и время, ограничение `NOT NULL`.
  - `type_event` — тип события (`char(1)`), значение по умолчанию — `'I'`, ограничение `NOT NULL`. Добавь проверочное ограничение `ch_type_event` со значениями `'I'`, `'U'`, `'D'`.
  - `row_id` — копия значения `person.id` (`bigint`), ограничение `NOT NULL`.
  - `name` — копия значения `person.name`.
  - `age` — копия значения `person.age`.
  - `gender` — копия значения `person.gender`.
  - `address` — копия значения `person.address`.

Создай триггерную функцию `fnc_trg_person_insert_audit`, которая будет обрабатывать операции `INSERT` и сохранять копию новой строки в таблице `person_audit`.

Создай триггер `trg_person_insert_audit` со следующими параметрами:

- `FOR EACH ROW`;
- `AFTER INSERT`;
- вызывает функцию `fnc_trg_person_insert_audit`.

После создания триггера выполни вставку новой записи в таблицу `person`.
